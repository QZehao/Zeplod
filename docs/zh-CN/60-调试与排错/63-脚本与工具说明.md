# 脚本与工具说明

本文档集中说明仓库 `scripts/` 目录下各脚本的作用、典型用法与注意事项。

脚本通过 **`project_layout.ps1` / `project_layout.sh` / `project_layout.py`** 自动识别两种仓库布局：

| 布局 | 识别条件 | 脚本路径 | 工作目录 |
|---|---|---|---|
| **framework** | 本仓库即框架本体（如 zeplod） | `scripts/` | 仓库根 |
| **app** | 上层 `CMakeLists.txt` 含 `add_subdirectory(framework)` | `framework/scripts/` | APP 仓库根 |

`zephyr_config.env` 始终位于 **framework 目录**（APP 模式下为 `framework/zephyr_config.env`）。可选在 APP 根目录放置 **`zephyr_app.env`**（见仓库根 `zephyr_app.env.template`）覆盖 `APP_PRJ_CONF`、`QEMU_CONF` 等。

## 1. 快速分组

| 分组 | 脚本 |
|---|---|
| 布局解析（内部） | `project_layout.ps1` / `project_layout.sh` / `project_layout.py` |
| 环境初始化 | `setup_env.ps1` / `setup_env.sh` / `setup_env.bat` |
| QEMU 仿真 | `run_qemu.ps1` |
| 主机测试 | `run_tests.ps1` / `run_tests.sh` |
| 主机 Sanitizer | `run_sanitizers.ps1` / `run_sanitizers.sh` |
| Twister | `run_twister.ps1` / `run_twister.sh` |
| 统一 QA 入口 | `qa.ps1` / `qa.sh` |
| 预检 | `preflight_host_tests.py` |
| 文档与检查 | `generate_docs.ps1` / `generate_docs.sh` / `lint_docs.py` / `check_encoding.py` / `check_script_docs.py` |
| 版本与发布 | `bump_version.py` / `package_release.ps1` / `package_release.sh` |
| 构建分析 | `build_all.ps1` / `build_all.bat` / `build_all.sh` / `analyze_map.ps1` / `analyze_map.sh` / `analyze_map.bat` |
| 配置与模块 | `module_config.py` / `proprietary_manage.ps1` / `proprietary_manage.sh` / `proprietary_manage.bat` |

## 2. 运行前提

1. 已准备 `zephyr_config.env`（由 `zephyr_config.env.template` 复制并填写；APP 仓库放在 **`framework/`** 下）。  
2. 已安装 Zephyr 开发工具（west、CMake、Python、Zephyr SDK）。  
3. 主机测试（`native_sim/native_posix`）建议在 Linux/WSL 运行。Windows 原生会被脚本拦截并给出提示。

### 2.1 两种仓库下的命令前缀

**框架本体（zeplod）**：

```powershell
.\scripts\setup_env.ps1
.\scripts\run_qemu.ps1
```

**基于 framework 的 APP（如 zephyr_gateway）**：

```powershell
.\framework\scripts\setup_env.ps1
.\framework\scripts\run_qemu.ps1
.\framework\scripts\qa.ps1 -Mode all
```

下文示例默认写 `scripts/`；APP 仓库请替换为 `framework/scripts/`。

## 3. 各脚本说明与用法

### 3.1 环境初始化

| 脚本 | 作用 | 用法 |
|---|---|---|
| `setup_env.ps1` | 读取 `zephyr_config.env`，设置 `ZEPHYR_BASE` 等变量并激活 venv（若配置） | `.\scripts\setup_env.ps1` |
| `setup_env.sh` | Linux/macOS/WSL 版本 | `source scripts/setup_env.sh` |
| `setup_env.bat` | Windows CMD 版本 | `scripts\setup_env.bat` |

### 3.2 QEMU 仿真（Windows）

| 脚本 | 作用 | 用法 |
|---|---|---|
| `run_qemu.ps1` | 加载环境、构建并 `west build -t run`；framework 模式用 `prj.conf;conf/targets/qemu.conf`，app 模式自动合并 `framework/prj.conf`、`*_prj.conf`、`framework/conf/targets/qemu.conf` | `.\scripts\run_qemu.ps1` |

参数：`-Board <板型>`（默认 `qemu_riscv32`）、`-BuildDir`、`-BuildOnly`、`-ListBoards`、`-NoPristine`、`-Target auto|framework|app`（`framework` 在 APP 仓库中仅构建 `framework/` 子目录）。

环境变量：`ZEPHYR_QEMU_CONF` 可覆盖 QEMU 配置片段。

依赖 **`zephyr_config.env`** 中的 **`QEMU_BIN_PATH`**。详见 **[14-QEMU仿真运行指南.md](../10-环境与构建/14-QEMU仿真运行指南.md)**。

### 3.3 主机测试

| 脚本 | 作用 | 用法 |
|---|---|---|
| `run_tests.ps1` | 自动初始化环境 + 预检 + 运行 `tests/` | `.\scripts\run_tests.ps1` |
| `run_tests.sh` | Linux/macOS/WSL 版本 | `./scripts/run_tests.sh` |

可用环境变量：
- `ZEPHYR_TEST_BOARD`：覆盖板型（默认自动选 `native_sim`，退回 `native_posix`）
- `ZEPHYR_TEST_CONF`：测试配置文件（默认 `prj.conf;prj_test_extensions.conf`；硬件冒烟可设为 `prj.conf`）
- `ZEPHYR_TEST_BUILD_DIR`：构建目录（默认 `build_tests`）

### 3.4 主机 Sanitizer

| 脚本 | 作用 | 用法 |
|---|---|---|
| `run_sanitizers.ps1` | 在主机上运行 ASan/UBSan | `.\scripts\run_sanitizers.ps1` |
| `run_sanitizers.sh` | Linux/macOS/WSL 版本 | `./scripts/run_sanitizers.sh` |

可用环境变量：
- `ZEPHYR_SANITIZER=asan|ubsan|asan-ubsan`（默认 `asan-ubsan`）
- `ZEPHYR_TEST_BOARD`
- `ZEPHYR_TEST_CONF`
- `ZEPHYR_SAN_BUILD_DIR`（默认 `build_sanitizers`）

### 3.5 Twister

| 脚本 | 作用 | 用法 |
|---|---|---|
| `run_twister.ps1` | 运行 Zephyr Twister（`tests/`） | `.\scripts\run_twister.ps1 -Platform native_sim` |
| `run_twister.sh` | Linux/macOS/WSL 版本 | `./scripts/run_twister.sh` |

可用环境变量：
- `ZEPHYR_TWISTER_PLATFORM`（默认 `native_sim`）
- `ZEPHYR_TWISTER_OUT_DIR`（默认 `twister-out`）

### 3.6 预检

| 脚本 | 作用 | 用法 |
|---|---|---|
| `preflight_host_tests.py` | 检查主机 OS、west 可用性、host 板支持，快速失败 | `python scripts/preflight_host_tests.py` |

### 3.7 统一 QA 入口

| 脚本 | 作用 | 用法 |
|---|---|---|
| `qa.ps1` | 统一运行测试/sanitizer/twister | `.\scripts\qa.ps1 -Mode all` |
| `qa.sh` | Linux/macOS/WSL 版本 | `./scripts/qa.sh all` |

模式：
- `test`
- `san`
- `twister`
- `all`

### 3.8 文档与编码检查

| 脚本 | 作用 | 用法 |
|---|---|---|
| `generate_docs.ps1` / `generate_docs.sh` | 生成 Doxygen 文档 | `.\scripts\generate_docs.ps1` / `./scripts/generate_docs.sh` |
| `lint_docs.py` | 文档规范检查（按脚本实现） | `python scripts/lint_docs.py` |
| `check_encoding.py` | 检查文本文件 UTF-8（无 BOM） | `python scripts/check_encoding.py` |
| `check_script_docs.py` | 校验脚本文档中提到的脚本是否真实存在 | `python scripts/check_script_docs.py` |

### 3.9 版本与发布

| 脚本 | 作用 | 用法 |
|---|---|---|
| `bump_version.py` | 更新版本号相关文件 | `python scripts/bump_version.py -h` |
| `package_release.ps1` / `package_release.sh` | 打包发布产物 | 查看脚本内帮助/参数 |

### 3.10 构建分析

| 脚本 | 作用 | 用法 |
|---|---|---|
| `build_all.ps1` / `build_all.bat` / `build_all.sh` | 多板型批量 `west build`（APP 模式从 APP 根目录构建） | `.\scripts\build_all.ps1 -Board nucleo_l4r5zi` |
| `analyze_map.ps1` / `.sh` / `.bat` | 分析 `zephyr.map` 内存占用 | `.\scripts\analyze_map.ps1` |

### 3.11 配置与模块管理

| 脚本 | 作用 | 用法 |
|---|---|---|
| `module_config.py` | 模块配置处理（按脚本定义） | `python scripts/module_config.py -h` |
| `proprietary_manage.ps1` / `.sh` / `.bat` | 启用/禁用商用模块目录与配置 | `.\scripts\proprietary_manage.ps1 -h` |

## 4. 常用命令速查

```powershell
# Windows PowerShell
.\scripts\setup_env.ps1
.\scripts\run_qemu.ps1
# Windows 单元测试 QEMU 仿真（见 14-QEMU §6）
$env:ZEPHYR_TEST_BOARD = 'qemu_riscv32'; .\scripts\run_tests.ps1
python scripts\preflight_host_tests.py
.\scripts\run_tests.ps1
.\scripts\run_sanitizers.ps1
.\scripts\run_twister.ps1 -Platform native_sim
python scripts\check_encoding.py
```

```bash
# Linux / WSL
source scripts/setup_env.sh
python scripts/preflight_host_tests.py
./scripts/run_tests.sh
./scripts/run_sanitizers.sh
./scripts/run_twister.sh
python scripts/check_encoding.py
```

## 5. 平台注意事项

- `native_sim/native_posix` 属于 POSIX 主机架构，不适合 Windows 原生环境直接跑。  
- 在 Windows 上请优先使用 **QEMU 仿真**：主应用 `run_qemu.ps1`，单元测试 `ZEPHYR_TEST_BOARD=qemu_riscv32` + `run_tests.ps1`（见 [14-QEMU仿真运行指南.md](../10-环境与构建/14-QEMU仿真运行指南.md) §6），或使用 WSL 跑主机 POSIX 测试。  
- CI 已接入脚本预检、编码检查、覆盖率门禁、sanitizers 和 twister。

## 6. APP 仓库可选配置（`zephyr_app.env`）

从零搭建 APP 仓库的完整步骤见 **[15-新建APP开发指南.md](../10-环境与构建/15-新建APP开发指南.md)**。

在 APP 根目录创建 `zephyr_app.env`（可复制 `zephyr_app.env.template`）：

```bash
APP_PRJ_CONF=gateway_prj.conf
QEMU_CONF=framework/prj.conf;gateway_prj.conf;framework/conf/targets/qemu.conf
```

未配置时，脚本会尝试自动发现 `*_prj.conf` 并拼接 `framework/conf/targets/qemu.conf`。

## 7. 高级环境变量

| 变量 | 作用 |
|---|---|
| `ZEPHYR_PROJECT_TARGET` | `auto` / `framework` / `app`，强制布局目标 |
| `ZEPHYR_APP_ROOT` | 强制 APP 根目录 |
| `ZEPHYR_QEMU_CONF` | 覆盖 QEMU `CONF_FILE` 片段 |
