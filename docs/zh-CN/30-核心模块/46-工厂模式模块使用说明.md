# 工厂模式模块使用说明（factory_mode）

本文说明可选 **`factory_mode`** 模块（`CONFIG_FACTORY_MODE_MODULE`）：产线进入/退出、GPIO 环回（Phase 4 stub）、校准数据暂存与提交到 `app_kv`，发布事件 **70**（`EVENT_FACTORY_STATE_CHANGED`）。

**要求：** `CONFIG_EVENT_MAX_TYPES > 70`

**相关**：[42-项目配置项说明.md](../40-应用开发/42-项目配置项说明.md) · [74-OTA与存储扩展指南.md](../70-发布与产品化/74-OTA与存储扩展指南.md)

---

## 1. 启用

产线固件：

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/factory_mode.conf
```

量产固件（默认关闭 factory）：

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/targets/production.conf
```

---

## 2. 状态机

| 状态 | 说明 |
|------|------|
| `INACTIVE` | 默认 |
| `ACTIVE` | 产测进行中 |
| `PASSED` | GPIO + 校准已提交 |
| `FAILED` | 提交 app_kv 失败 |

典型流程：`enter` → `gpio_loopback` → `set_calibration` × N → `finalize_pass`。

---

## 3. Shell 命令

启用 `CONFIG_FACTORY_MODE_SHELL`（依赖 `CONFIG_SHELL`）：

| 命令 | 说明 |
|------|------|
| `factory enter` | 进入 ACTIVE |
| `factory exit` | 退出并清空暂存 |
| `factory status` | 打印状态 |
| `factory gpio` | GPIO 环回（stub 同步通过） |
| `factory cal set <k> <v>` | 暂存校准项 |
| `factory cal get <k>` | 读取暂存项 |
| `factory finalize` | 写入 `factory.cal.<k>` 并标记 PASSED |

---

## 4. API

```c
int factory_mode_enter(void);
int factory_mode_exit(void);
int factory_mode_get_state(factory_status_t* out);
int factory_mode_run_gpio_loopback(void);
int factory_mode_set_calibration(const char* key, const char* value);
int factory_mode_get_calibration(const char* key, char* out, size_t out_len);
int factory_mode_finalize_pass(void);
```

`finalize_pass` 需要 `CONFIG_APP_KV_ENABLE=y`，将校准写入键 `factory.cal.<name>`。

`factory_mode_get_state()` 在 `FAILED` 时通过 `error_code` 返回最近一次失败码（如 `APP_ERR_KV_FULL`），其它状态为 0。

## 5. 测试

```bash
west build -b native_sim tests/ --build-dir build_tests_factory -p always \
  -- -DCONF_FILE="prj.conf;prj_factory.conf"
west build -t run --build-dir build_tests_factory

# 无 app_kv（finalize 返回 APP_ERR_DISABLED）
west build -b native_sim tests/ --build-dir build_tests_factory_no_kv -p always \
  -- -DCONF_FILE="prj.conf;prj_factory_no_kv.conf"
west build -t run --build-dir build_tests_factory_no_kv
```

用例：`tests/test_factory_mode.c`（完整产测流、事件订阅、失败路径与边界）；无 app_kv 见 `tests/prj_factory_no_kv.conf` + `test_factory_mode_no_kv.c`。

## 6. 量产注意

- `production.conf` 默认 `CONFIG_FACTORY_MODE_MODULE=n`；产线与量产应使用不同叠加链。
- 真实 GPIO 环回：关闭 `CONFIG_FACTORY_MODE_GPIO_STUB` 并实现硬件后端（Phase 5 扩展）。
- **`finalize_pass` 非原子**：多键提交时若中途 `app_kv_set` 失败，已成功写入的 `factory.cal.*` **不会回滚**，状态变为 `FAILED`；`factory_mode_get_state()` 的 `error_code` 保留失败码。产线需人工处理或重新 `exit` → `enter` 后重跑。
- **并发**：`finalize_pass` 写入 app_kv 时会短暂释放模块锁；产线默认单线程 Shell/API 调用，勿与其它线程并发修改校准表。
