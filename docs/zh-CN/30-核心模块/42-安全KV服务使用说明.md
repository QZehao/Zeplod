# 安全 KV 服务使用说明（sys_secure_kv）

本文说明可选 **`sys_secure_kv`** 服务（`CONFIG_SYS_SECURE_KV_ENABLE`）：在 RAM 中存储敏感二进制值，Phase 2 使用软件 keystream 加解密；量产请换 PSA / AES-GCM 后端。

**相关**：[36-系统服务使用说明.md](36-系统服务使用说明.md) · [75-安全与密钥管理说明.md](../70-发布与产品化/75-安全与密钥管理说明.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/secure_kv.conf
```

密钥来自 Kconfig `CONFIG_SYS_SECURE_KV_KEY_HEX`（32 个十六进制字符 = 16 字节）。**勿将量产密钥提交进仓库**；开发默认值仅用于本地与 CI。

---

## 2. 与 app_kv 区别

| 项 | `app_kv` | `sys_secure_kv` |
|----|----------|-----------------|
| 值类型 | 字符串 | 二进制 |
| 加密 | 无 | Phase 2 keystream（可替换） |
| 持久化 | 可选 NVS | 仅 RAM |

敏感配置（Wi-Fi PSK、令牌）建议用 `sys_secure_kv`；普通字符串配置仍用 `app_kv`。

---

## 3. API

```c
int sys_secure_kv_init(void);
int sys_secure_kv_set(const char* key, const uint8_t* plain, size_t plain_len);
int sys_secure_kv_get(const char* key, uint8_t* out, size_t out_len, size_t* out_written);
int sys_secure_kv_remove(const char* key);
bool sys_secure_kv_has(const char* key);
void sys_secure_kv_clear(void);
```

`init` 幂等，由 `SYS_INIT`（`APP_INIT_PRIO_SYS_SECURE_KV`）自动调用。

---

## 4. 测试

```bash
west build -b native_sim tests/ --build-dir build_tests_secure_kv -p always \
  -- -DCONF_FILE="prj.conf;prj_sys_secure_kv.conf"
west build -t run --build-dir build_tests_secure_kv
```

用例：`tests/test_sys_secure_kv.c`（round-trip、删除、清空）。

---

## 5. 量产注意

- 替换 `CONFIG_SYS_SECURE_KV_KEY_HEX` 为产线注入或安全元件派生密钥
- Phase 2 软件 keystream **非**认证级加密；威胁模型见 [75-安全与密钥管理说明.md](../70-发布与产品化/75-安全与密钥管理说明.md)
