# 系统诊断服务使用说明（sys_diag）

本文说明可选 **`sys_diag`** 服务（`CONFIG_SYS_DIAG_ENABLE`）：聚合堆内存、事件队列与模块管理器健康快照，并提供 Shell `diag dump`。

**相关**：[36-系统服务使用说明.md](36-系统服务使用说明.md) · [产品化扩展路线图](../../superpowers/plan/2026-06-13-product-framework-expansion-roadmap.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/diag.conf
```

`conf/features/diag.conf` 默认同时打开：

- `CONFIG_SYS_DIAG_ENABLE=y`
- `CONFIG_SYS_MEMORY_ENABLE=y`
- `CONFIG_MODULE_MANAGER=y`
- `CONFIG_EVENT_SYSTEM=y`

未启用的子系统对应快照字段为 **0**（不会崩溃）。

---

## 2. 快照结构

```c
typedef struct {
    uint32_t heap_free_bytes;
    uint32_t heap_used_bytes;
    uint32_t event_queue_depth;
    uint32_t event_queue_capacity;
    uint32_t event_dropped_count;
    uint32_t module_count;
    uint32_t module_running_count;
    uint32_t module_error_count;
    uint32_t uptime_ms;
} sys_diag_snapshot_t;
```

| 字段 | 数据来源 |
|------|----------|
| `heap_*` | `sys_mem_get_heap_size()` / `sys_mem_get_free_size()` |
| `event_queue_*` | `event_get_statistics()` + `CONFIG_EVENT_QUEUE_SIZE` |
| `event_dropped_count` | `event_get_statistics()` |
| `module_*` | `module_manager_get_stats()` |
| `uptime_ms` | `k_uptime_get_32()` |

---

## 3. API

```c
int sys_diag_init(void);
int sys_diag_collect(sys_diag_snapshot_t *out);
int sys_diag_format(const sys_diag_snapshot_t *snap, char *buf, size_t buf_len);
int sys_diag_export_json(const sys_diag_snapshot_t *snap, char *buf, size_t buf_len);
```

- `sys_diag_init()` 由 `SYS_INIT(POST_KERNEL, APP_INIT_PRIO_SYS_DIAG)` 自动调用，可重复调用。
- `sys_diag_format()` 示例输出：

```
diag: heap free=8192 used=1024 evt=3/64 drop=0 mod=2 run=2 err=0 up=12345ms
```

---

## 4. Shell

在 `CONFIG_SHELL=y` 且 `CONFIG_SYS_DIAG_ENABLE=y` 时：

```
diag dump
```

命令在 `src/services/sys_diag.c` 注册，**不依赖** `CONFIG_APP_ENABLE_SHELL`。
