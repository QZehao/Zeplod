# 远程运维模块使用说明（remote_ops）

本文说明可选 **`remote_ops`** 模块（`CONFIG_REMOTE_OPS_MODULE`）：将 `sys_diag` 快照序列化为 JSON 并经可插拔后端导出；Phase 5 为 **null 后端**（RAM 缓存），mcumgr 传输见后续 `conf/targets/` 叠加。事件 **71**。

**要求：** `CONFIG_EVENT_MAX_TYPES > 71`、`CONFIG_SYS_DIAG_ENABLE=y`

**相关**：[40-系统诊断服务使用说明.md](40-系统诊断服务使用说明.md) · [48-功能开关模块使用说明.md](48-功能开关模块使用说明.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/remote_ops.conf
```

规模化三合一：

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/scale.conf
```

---

## 2. API

```c
int remote_ops_export_diag(void);
int remote_ops_get_last_export(char* out, size_t out_len);
int remote_ops_get_stats(remote_ops_status_t* out);
```

`export_diag` 内部调用 `sys_diag_export_json()`，再经后端 `export_diag` 回调发送。模块须处于 **RUNNING** 状态。

`get_last_export` 依赖后端实现 `get_last_export` 回调（null 后端提供 RAM 读回；mcumgr 等无缓存后端可返回 `-ENOTSUP`）。

---

## 3. Shell

```
remote export
```

打印最近一次导出的 JSON 行。

---

## 4. 测试

```bash
west build -b native_sim tests/ --build-dir build_tests_remote_ops -p always \
  -- -DCONF_FILE="prj.conf;prj_remote_ops.conf"
west build -t run --build-dir build_tests_remote_ops
```
