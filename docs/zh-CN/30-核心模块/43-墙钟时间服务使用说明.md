# 墙钟时间服务使用说明（sys_time）

本文说明可选 **`sys_time`** 服务（`CONFIG_SYS_TIME_ENABLE`）：在设置基准 Unix 毫秒时间后，用 `k_uptime` 推算当前墙钟；Phase 3 为软件实现，SNTP / RTC 后端可在后续 `conf/targets/` 叠加。

**相关**：[36-系统服务使用说明.md](36-系统服务使用说明.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/sys_time.conf
```

或与其他联网骨架一并启用：

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/net_stub.conf
```

---

## 2. 初始化

`sys_time_init()` 由 `SYS_INIT`（`APP_INIT_PRIO_SYS_TIME`）自动调用，幂等。

在未调用 `sys_time_set_unix_ms()` 或 SNTP 同步前，`sys_time_is_valid()` 为 `false`，`sys_time_get_unix_ms()` 返回 `APP_ERR_TIME`（未设置）或 `APP_ERR_INVALID_PARAM`（参数无效）。

---

## 3. API

```c
int sys_time_init(void);
int sys_time_set_unix_ms(int64_t unix_ms);
int sys_time_get_unix_ms(int64_t* out_unix_ms);
bool sys_time_is_valid(void);
void sys_time_invalidate(void);  /* 测试用 */
```

---

## 4. 测试

```bash
west build -b native_sim tests/ --build-dir build_tests_sys_time -p always \
  -- -DCONF_FILE="prj.conf;prj_sys_time.conf"
west build -t run --build-dir build_tests_sys_time
```

用例：`tests/test_sys_time.c`（设置/读取、无效态、负时间戳拒绝）。

---

## 5. Phase 4 衔接

- SNTP 客户端同步后调用 `sys_time_set_unix_ms()`
- 硬件 RTC 可在启动早期设置基准，再由 `sys_time` 统一对外 API
