# 配网模块使用说明（provisioning_module）

本文说明可选 **`provisioning_module`**（`CONFIG_PROVISIONING_MODULE`）：设备身份与配网状态机，通过事件 **61**（`EVENT_PROVISIONING_STATE_CHANGED`）广播状态。Phase 3 为 **stub**（`begin` 立即成功），凭据持久化与 BLE 配网留 Phase 4。

**要求：** `CONFIG_EVENT_MAX_TYPES > 61`

**相关**：[44-连接管理模块使用说明.md](44-连接管理模块使用说明.md) · [42-安全KV服务使用说明.md](42-安全KV服务使用说明.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/provisioning.conf
```

设备 ID 默认来自 Kconfig：

```
CONFIG_PROVISIONING_DEVICE_ID="zeplod-dev-00000001"
```

量产应在板级 overlay 或工厂流程中覆盖，勿所有 SKU 共用同一默认值。

---

## 2. 状态机

| 状态 | 说明 |
|------|------|
| `UNPROVISIONED` | 初始 / reset 后 |
| `IN_PROGRESS` | 配网进行中（stub 瞬时） |
| `PROVISIONED` | 已完成 |
| `ERROR` | 失败（后续后端使用） |

事件负载：`provisioning_status_t`（state + error_code）。

---

## 3. API

```c
int provisioning_module_begin(const provisioning_credentials_t* creds);
int provisioning_module_reset(void);
int provisioning_module_get_state(provisioning_state_t* out_state);
int provisioning_module_get_device_id(char* out, size_t out_len);
```

`provisioning_credentials_t` 含 `ssid` / `psk` 指针；Phase 3 stub 忽略内容。敏感 PSK 建议 Phase 4 写入 `sys_secure_kv`。

---

## 4. 测试

```bash
west build -b native_sim tests/ --build-dir build_tests_provisioning -p always \
  -- -DCONF_FILE="prj.conf;prj_provisioning.conf"
west build -t run --build-dir build_tests_provisioning
```

用例：`tests/test_provisioning_module.c`（begin/reset、device_id、生命周期）。
