# 连接管理模块使用说明（connectivity_module）

本文说明可选 **`connectivity_module`**（`CONFIG_CONNECTIVITY_MODULE`）：抽象链路 up/down，通过事件 **60**（`EVENT_CONNECTIVITY_STATE_CHANGED`）广播状态。Phase 3 默认 **null 后端**（同步模拟连接），真实 Wi-Fi / 以太网在 Phase 4 实现。

**要求：** `CONFIG_EVENT_MAX_TYPES > 60`

**相关**：[32-模块系统详细使用说明.md](32-模块系统详细使用说明.md) · [45-配网模块使用说明.md](45-配网模块使用说明.md)

---

## 1. 启用

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/connectivity.conf
```

三合一 stub：

```bash
west build -b <board> . -- -DEXTRA_CONF_FILE=conf/features/net_stub.conf
```

---

## 2. 状态与事件

| 字段 | 说明 |
|------|------|
| `connectivity_state_t` | DOWN / CONNECTING / UP / ERROR |
| `connectivity_link_type_t` | NONE / WIFI / ETHERNET / BLE |
| 事件负载 | `connectivity_status_t`（state + link_type + error_code） |

---

## 3. API

模块生命周期遵循 `module_interface_t`；业务侧常用：

```c
int connectivity_module_connect(connectivity_link_type_t link_type);
int connectivity_module_disconnect(void);
int connectivity_module_get_state(connectivity_status_t* out);
```

`connectivity_module_connect()` 须模块已 **start**（`MODULE_STATUS_RUNNING`）；在 null 后端下同步进入 `CONNECTIVITY_STATE_UP`；已连接时再次 connect 幂等返回 0。

---

## 4. 后端扩展

实现 `connectivity_backend_ops_t`（见 `connectivity_backend.h`），通过 Kconfig `CONFIG_CONNECTIVITY_BACKEND_*` 选择后端并在 `connectivity_module_init` 中挂接（Phase 3 默认 null）。

---

```bash
  -- -DCONF_FILE="prj.conf;prj_connectivity.conf"
west build -t run --build-dir build_tests_connectivity
```
